# Agent 记录约定

当用户明确要求“记录问题”或“写成 md”时，优先把问题记录到 `notes/` 目录下。

## 约定

- 新问题记录优先创建在 `notes/` 下。
- 文件名要能直接表达问题主题，避免使用含糊的通用名。
- 记录内容只写结论、现象、旧实现状态、为什么先不处理，以及后续建议。
- 只要问题里存在需要业务、KB 维护方或公司内部确认的口径，必须单独写一个 `需要与公司确认` 或同义小节，明确列出待确认项。
- 如果同一问题已经有记录文件，优先更新原文件，不重复新建。

## 适用范围

- 适用于当前这个 `document-verification-master` 工作区。
- 适用于你明确要求“记录”“存档”“写 md”的问题。
- 不适用于普通分析回复，普通回复仍按正常对话输出。
