# Repository Agent Rules

This repository uses `notes/` as the canonical location for issue records.

## Recording rule

- When the user explicitly asks to record a problem, write it as a Markdown note under `notes/`.
- Use a descriptive filename that reflects the topic.
- If the same problem already has a note, update that file instead of creating a duplicate.
- Keep the note focused on the problem statement, observed behavior, existing implementation status, and the decision to defer or fix.

## Current convention

- `notes/agent.md` documents the recording convention itself.
- `notes/phase_error_iq_offset_kb_gap.md` is an example of the current issue record format.
