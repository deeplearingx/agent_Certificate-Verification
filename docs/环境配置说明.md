#### 方式一：使用 venv
```bash

# 激活虚拟环境

venv\Scripts\activate

source venv/bin/activate
```

```bash

conda create -n doc-verification python=3.10

# 激活环境
conda activate doc-verification
```

## 🔐 环境变量配置

### 配置文件

项目使用 `.env` 文件管理环境变量。请复制 `.env.example` 为 `.env`：

```bash
# Windows
copy .env.example .env

# Linux/macOS
cp .env.example .env
```

### 环境变量说明

#### DeepSeek API 配置
```bash
# DeepSeek API Key（必填）
DEEPSEEK_API_KEY=your_api_key_here

# DeepSeek API 地址
DEEPSEEK_API_BASE=https://api.deepseek.com/v1

# 使用的模型
DEEPSEEK_MODEL=deepseek-chat

# API 温度参数 (0-1)
DEEPSEEK_TEMPERATURE=0.1

# 最大 Token 数
DEEPSEEK_MAX_TOKENS=2048
```

#### 检索配置
```bash
# 检索结果数量
TOPK=50

# 批处理大小
BATCH_SIZE=5

# 最大并发工作线程数
MAX_WORKERS=5
```

#### 模型路径配置
```bash
# 嵌入模型路径
EMBED_MODEL_PATH=./models
```

#### 向量数据库配置
```bash
# CNAS 数据库路径
CNAS_DB_DIR=./vector_db/cnas_calibration

# 温度数据库路径
TEMPERATURE_DB_DIR=./vector_db/temperature

# 通用周期数据库路径
GENERAL_CYCLE_DB_DIR=./vector_db/general_cycle

# 华为周期数据库路径
HUAWEI_CYCLE_DB_DIR=./vector_db/huawei_cycle

# 地址数据库路径
ADDRESS_DB_DIR=./vector_db/address

# 数据库集合名称
CNAS_COLLECTION=calibration_data
ADDRESS_COLLECTION=calibration_address
```

#### 核验配置
```bash
# 默认校准周期
DEFAULT_CYCLE=12个月

# 是否使用 LLM 进行参数核验
USE_LLM_VERIFICATION=true

# 是否使用 LLM 进行地址核验
USE_LLM_LOCATION_CHECK=true

# 必须匹配阈值
MUST_MATCH_THRESHOLD=0.45

# 可选匹配阈值
OPTIONAL_MATCH_THRESHOLD=0.45

# 地址核验 LLM 温度
LOCATION_LLM_TEMPERATURE=0.0

# 地址核验 LLM 最大 Token
LOCATION_LLM_MAX_TOKENS=256
```

#### 文件路径配置
```bash
# 本地 PDF 文件目录
LOCAL_PDF_DIR=./local_pdf

# 本地 Markdown 文件目录
LOCAL_MD_DIR=./local_md

# 本地 JSON 文件目录
LOCAL_JSON_DIR=./local_json

# 最终报告输出目录
FINAL_REPORTS_DIR=./final_reports

# 中间报告目录
REPORTS_DIR=./reports
```

## 🚀 快速开始

### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 配置环境变量
```bash
# 复制示例配置
cp .env.example .env

# 编辑 .env，填入你的 API Key
DEEPSEEK_API_KEY=sk-xxxxxxxxxx
```

### 3. 准备数据
```bash
# 将需要核验的 PDF 证书放入 local_pdf 目录
mkdir -p local_pdf
# 复制 PDF 文件到该目录
```

### 4. 运行应用
```bash
# 方式一：使用 Streamlit Web 界面
streamlit run app.py

# 方式二：使用命令行脚本
python main_pipeline.py
```

## 🔍 常见问题

### Q1: 安装依赖时出现错误？

**A:** 尝试以下解决方案：
```bash
# 升级 pip
python -m pip install --upgrade pip

# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 逐个安装有问题的包
pip install <package_name>
```

### Q2: 找不到模块？

**A:** 确保在项目根目录下运行：
```bash
# 方式一：切换到项目根目录
cd /path/to/document-verification

# 方式二：设置 PYTHONPATH
export PYTHONPATH=/path/to/document-verification:$PYTHONPATH  # Linux/macOS
set PYTHONPATH=C:\path\to\document-verification;%PYTHONPATH%  # Windows
```

### Q3: 如何使用 GPU 加速？

**A:** 安装对应 CUDA 版本的 PyTorch：
```bash
# 检查 CUDA 版本
nvidia-smi

# 安装对应 CUDA 版本的 PyTorch
# 访问 https://pytorch.org/get-started/locally/ 获取最新命令
```

### Q4: 如何运行测试？

**A:** 使用 pytest 运行测试：
```bash
# 运行所有测试
pytest tests/

# 运行特定测试
pytest tests/test_param_uncertainty_kinds.py

# 生成测试覆盖率报告
pytest tests/ --cov=. --cov-report=html
```

### Q5: 在哪里可以获取帮助？

**A:** 查看项目文档：
- [param_check_代码分析.md](./param_check_代码分析.md) - 参数核验模块详细分析
- 查看项目根目录下的 README.md（如果有）
