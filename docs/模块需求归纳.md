# 模块需求归纳

本文按当前 `langchain_app/` 主线整理各模块需求，用于分工、验收和后续迭代对齐。每个模块都按“职责、输入、输出、代码位置、验收标准”归纳。

## 1. PDF 解析与结构化模块

### 职责

- 将证书 PDF 转换为 Markdown。
- 将 Markdown 结构化为 JSON。
- 保留证书基础字段、环境字段、地点字段、周期字段、依据字段和参数表字段。
- 对参数表输出标准化字段，方便后续参数核验直接读取。
- 标记解析风险，例如字段缺失、表格错列、单位继承、跨页表格、非 CNAS 证书。

### 输入

```text
PDF 文件路径
AppConfig 配置
PipelineHooks / Graph hooks
stop_event
可选 llm_client
```

### 输出

```text
local_md/<pdf_stem>.md
local_json/<pdf_stem>.json
```

JSON 至少需要包含：

- `CNAS`
- `证书编号`
- `委托单位`
- `委托方地址`
- `仪器名称`
- `型号规格`
- `制造厂`
- `机身号` / `序列号` / `管理号`
- `接收日期`
- `校准日期`
- `温度`
- `相对湿度`
- `校准地点`
- `建议校准周期`
- `校准依据`
- `依据参数_中间数据`

参数表行建议包含：

- `项目名称`
- `数据明细`
- `__normalized_fields`
- `__parameter_contract`
- `__parser_meta`

### 代码位置

- `langchain_app/services/parsing.py`
- `langchain_app/services/md_parser_pipeline.py`
- `langchain_app/services/generic_md_parser_template.py`
- `langchain_app/graph/nodes/parse_pdf.py`
- `langchain_app/graph/nodes/parse_json.py`
- `pdf_md.py`
- `md_parser_no_llm.py`

### 验收标准

- 正常 CNAS PDF 能稳定输出 MD 和 JSON。
- 非 CNAS PDF 能被识别并提前进入跳过路径。
- JSON 字段缺失时不能乱填猜测值，应保留空值或风险说明。
- 参数表能够被参数核验模块收集到。
- `__parameter_contract.schema_version` 与 `contracts.py` 保持一致。
- 参数表原始字段必须可追溯，不能只保留标准化结果。

## 2. RAG 与向量数据库模块

### 职责

- 维护 Chroma 向量库读取、检索、fallback 检索和诊断信息。
- 支持 CNAS 能力库、温湿度库、地址库、通用周期库、华为周期库。
- 给核验模块提供可解释的检索依据。
- 区分“查不到”和“系统异常”，避免把知识库问题误判为证书不合格。
- 维护一次性建库脚本和向量库字段说明。

### 输入

```text
query 文本
AppConfig 配置
topk
可选 filter_condition
可选 embedding_function / embedder_obj
```

典型 query 来源：

- 校准依据
- 仪器名称
- 型号规格
- 校准地点
- 参数名称
- 测量点
- 周期字段

### 输出

当前兼容输出主要是 `Document` 或字典列表。推荐统一为：

```json
{
  "page_content": "知识库条目正文",
  "metadata": {
    "collection": "collection_name",
    "db_dir": "vector_db/...",
    "source": "来源文件或条目编号",
    "distance": 0.0,
    "score": 1.0,
    "file_code": "JJG/JJF/GJB ...",
    "standard_name": "规程名称",
    "instrument_name": "仪器名称",
    "measured": "被测量/能力项",
    "measure_range_text": "测量范围",
    "error_limit_text": "最大允许误差或允差",
    "uncertainty_text": "不确定度",
    "address": "标准地址",
    "temperature_requirement": "温度要求",
    "humidity_requirement": "湿度要求",
    "cycle_requirement": "校准周期要求"
  }
}
```

诊断信息建议包含：

```json
{
  "ok": true,
  "diagnostic": {
    "code": "OK/DB_MISSING/COLLECTION_MISSING/EMBEDDING_UNAVAILABLE/EMPTY_COLLECTION/NO_SAME_BASIS/LOW_SIMILARITY/METADATA_INCOMPLETE",
    "message": "诊断说明",
    "fallback_used": false
  }
}
```

### 代码位置

主线检索：

- `langchain_app/core/vector_db.py`
- `langchain_app/core/embedding_loader.py`
- `langchain_app/retrieval/cnas.py`
- `langchain_app/retrieval/temperature.py`
- `langchain_app/retrieval/address.py`
- `langchain_app/retrieval/cycle.py`
- `langchain_app/checks/parameter/retrieval.py`

建库脚本：

- `scripts_by_part/03_vector_db_builders/`

### 验收标准

- 每个库能说明 collection 名称、db 路径和 metadata 字段。
- 检索结果必须包含来源、距离或分数。
- 向量库路径不存在时返回系统诊断，不返回业务 FAIL。
- collection 缺失、embedding 不可用、空库、低相似度都要能区分。
- 参数核验能拿到同规程号、同能力项的候选条目。
- 环境、地点、周期核验能拿到可解释的依据片段。

## 3. 基础核验模块

基础核验包括完整性、环境、地点、周期四类。

环境、地点、周期的详细比较对象、业务规则、当前实现差异和交付契约见 [`基础核验模块需求说明.md`](./基础核验模块需求说明.md)。

### 3.1 完整性核验

#### 职责

- 检查证书基础字段是否完整。
- 检查关键字段是否为空、缺失或明显异常。
- 对非 CNAS 或字段严重缺失证书给出 REVIEW / ERROR。

#### 输入

```text
local_json/<pdf_stem>.json
AppConfig
可选 llm_client
```

#### 输出

Markdown 报告片段，包含：

- 缺失字段列表
- 已识别字段列表
- 判定状态
- 原因说明

#### 代码位置

- `langchain_app/checks/integrity.py`
- `langchain_app/graph/nodes/integrity_check.py`

#### 验收标准

- 能区分字段缺失和字段值不合格。
- 不因为解析缺失直接给业务 FAIL，应优先 REVIEW。
- 报告能列出具体缺失字段。

### 3.2 环境核验

#### 职责

- 校验证书温度、湿度是否满足依据或知识库要求。
- 支持温湿度字段来自首页或内页。
- 引用温湿度向量库检索结果。

#### 输入

```text
JSON 环境字段
仪器名称
校准依据
temperature vector db 检索结果
```

#### 输出

Markdown 报告片段，包含：

- 证书温湿度
- 知识库要求
- 检索来源
- PASS / FAIL / REVIEW / ERROR

#### 代码位置

- `langchain_app/checks/environment.py`
- `langchain_app/retrieval/temperature.py`
- `langchain_app/graph/nodes/environment_check.py`

#### 验收标准

- 温湿度可解析时能做范围判断。
- 检索不到要求时应进入 REVIEW，不直接 FAIL；当前实现会输出“待定”，后续需统一状态。
- 向量库异常时进入 ERROR。
- 首页与内页温湿度一致性应纳入环境核验；当前代码暂在周期模块中执行。

### 3.3 地点核验

#### 职责

- 校验证书中的校准地点是否与地址库或认可范围匹配。
- 支持地址文本模糊匹配和向量检索。
- 输出匹配依据和相似度。

#### 输入

```text
校准地点
委托方地址
仪器名称
address vector db
CNAS vector db
```

#### 输出

Markdown 报告片段，包含：

- 证书地点
- 命中地址
- 匹配分数或 distance
- 判定状态

#### 代码位置

- `langchain_app/checks/location.py`
- `langchain_app/retrieval/address.py`
- `langchain_app/retrieval/cnas.py`
- `langchain_app/graph/nodes/location_check.py`

#### 验收标准

- 能解释匹配到哪个地址或专业室。
- 同规程下无法确认是否带 `*` 时应进入 REVIEW 或 ERROR，不应静默当作不带 `*`。
- 不带 `*` 的地点必须命中授权地址库；带 `*` 的现场地点可接受足够具体的地点描述。
- 地址库异常时进入 ERROR。

### 3.4 周期核验

#### 职责

- 判断建议校准周期是否合理。
- 区分通用周期库和华为周期库。
- 支持按仪器名称、客户类型或特殊规则检索。

#### 输入

```text
仪器名称
建议校准周期
委托单位或客户信息
general_cycle vector db
huawei_cycle vector db
```

#### 输出

Markdown 报告片段，包含：

- 证书建议周期
- 知识库建议周期
- 命中来源
- 判定状态

#### 代码位置

- `langchain_app/checks/cycle.py`
- `langchain_app/retrieval/cycle.py`
- `langchain_app/graph/nodes/cycle_check.py`

#### 验收标准

- 能区分通用周期和特殊客户周期。
- 周期字段缺失时进入 REVIEW。
- 接收日期必须早于或等于校准日期。
- 建议周期超过 1 年时，必须有依据文件、规程或说明书支持。
- 知识库不可访问时进入 ERROR。
- 当前实现无周期库命中时会使用默认周期兜底，后续需在报告中显式标注 REVIEW 或“默认兜底”证据。

## 4. 参数核验模块

更细的比较对象、比较规则和 PASS/FAIL/REVIEW 口径见 [`参数核验模块需求说明.md`](./参数核验模块需求说明.md)。

### 职责

- 核验证书参数表中的测量范围、误差、允许误差和不确定度。
- 核验证书校准项目、范围和不确定度是否落在《认可的校准和测量能力表》内。
- 后续需核验超出能力表范围时的 `▽` 标识，以及内页 `P/F/P*` 结论是否符合 JJF 1094 判定逻辑。
- 根据仪器类型选择不同规则包。
- 复用通用数值解析、单位解析、范围判定、误差判定和不确定度判定。
- 对复杂参数语义进行识别和候选 KB 选择。
- 对单条参数无法确定的情况进入 REVIEW，而不是强行 PASS / FAIL。
- 对向量库访问失败、依据代号无法解析、KB 完全没有同规程条目等系统/数据前置条件失败，进入 ERROR。

### 输入

```text
JSON 参数表
校准依据
仪器名称
CNAS 能力库检索结果
AppConfig
可选 llm_client
可选 embedder_obj
```

### 输出

Markdown 参数核验报告，核心表格包含：

- 序号
- 点位
- 测量点
- 测试条件
- KB 编号
- KB 条目
- 证书匹配项
- 范围
- 证书误差
- 允许误差
- 证书 U
- KB_U
- 判定
- 说明

### 规则划分

当前已经按仪器规则包划分：

- `time_frequency.py`：时间频率类，包括计数器、时间间隔、秒表、频率标准、GNSS。
- `rf_microwave.py`：射频/微波类，包括信号源、相噪、功率、调制质量等。
- `oscilloscope_function.py`：示波器、函数/任意波形发生器。
- `generic.py`：无法识别仪器时的通用 fallback。
- `common.py`：所有仪器复用的通用核验阶段。
- `router.py`：根据仪器名称、依据号、PDF 分类选择规则包。

规则包通过 `semantic_whitelist` 控制当前仪器允许启用的参数规则。例如：

- 时间频率类：`frequency_accuracy`、`frequency_range`、`period_accuracy`、`period_range`、`count_accuracy`、`input_sensitivity`、`reference_oscillator`
- 射频微波类：`frequency_accuracy`、`power_accuracy`、`phase_noise`、`modulation_quality`、`spectral_purity`、`dynamic_range`、`vswr_accuracy`、`impedance_accuracy`
- GNSS：`frequency_accuracy`、`power_accuracy`、`phase_noise`、`modulation_quality`、`spectral_purity`、`dynamic_range`、`cnr_consistency`、`position_consistency`、`reference_oscillator`

### 代码位置

- `langchain_app/checks/parameter/parameter.py`
- `langchain_app/checks/parameter/contracts.py`
- `langchain_app/checks/parameter/semantic.py`
- `langchain_app/checks/parameter/selector.py`
- `langchain_app/checks/parameter/validator.py`
- `langchain_app/checks/parameter/rules.py`
- `langchain_app/checks/parameter/reporter.py`
- `langchain_app/checks/parameter/planner.py`
- `langchain_app/checks/parameter/profiles/`
- `langchain_app/checks/parameter/rule_sets/`
- `langchain_app/checks/parameter/PROFILE_ARCHITECTURE.md`

### 验收标准

- 能说明每个参数选中了哪条 KB。
- 能说明范围、误差、不确定度分别如何判定。
- 不同仪器类型能走不同规则包。
- 不属于当前仪器规则包的 semantic target 会进入 REVIEW / unknown，不误套其他仪器规则。
- KB 缺项和证书不合格必须区分。
- 同依据下单条参数 KB 缺项应为 REVIEW；整个依据无同规程 KB 或索引异常应为 ERROR。
- 超出能力表项目或范围时，应检查证书是否有 `▽` 标识；当前尚未完整落地。
- 有技术指标的测量点，应按 JJF 1094 复核证书内页 `P/F/P*` 结论；当前尚未完整落地。
- 标准器精度比和量程覆盖核验属于后续扩展项，当前参数核验主线尚未覆盖。
- LLM 不可用不能直接导致业务 FAIL。

## 10. 统一状态要求

所有核验模块统一使用：

| 状态 | 含义 |
| --- | --- |
| PASS | 证书内容满足规则 |
| FAIL | 证书内容明确不满足规则 |
| REVIEW | 信息不足、语义不确定、知识库缺项或需要人工复核 |
| ERROR | 系统、配置、解析或知识库异常 |

关键原则：

- 知识库缺项不等于证书 FAIL。
- 解析字段缺失不等于证书 FAIL。
- LLM 不可用不等于证书 FAIL。
- FAIL 必须有明确证书值、依据值和规则说明。
- REVIEW 必须说明人工复核原因。
