# 智能文档核验系统 - 技术方案文档（简化版）

## 一、项目概述

本项目是一个基于人工智能的CNAS校准证书自动化核验系统，实现了从PDF上传到生成核验报告的完整流程。系统结合了OCR技术、大语言模型（LLM）和向量数据库检索，能够对校准证书进行完整性、准确性和合规性检查。

---

## 三、技术方案详解

### 3.1 整体技术架构

系统采用分层架构设计，从上到下依次为：

1. **前端展示层**：Streamlit Web界面，提供用户交互
2. **业务编排层**：流水线编排 + 统一配置管理
3. **文档解析层**：PDF转Markdown + Markdown转JSON
4. **智能核验层**：五大核验模块（完整性、环境、地点、周期、参数）
5. **数据检索层**：Chroma DB向量数据库 + bge-m3嵌入模型
6. **LlamaIndex集成层**：LLM通信封装 + 向量检索增强
7. **AI模型层**：DeepSeek LLM API + 本地bge-m3嵌入模型

---

### 3.2 各流程技术实现详解

#### 流程一：PDF → MD 转换（MinerU OCR）

**技术手段**：
- **OCR引擎**：MinerU混合引擎
- **解析方式**：PDF文本提取 + OCR识别双重方式
- **语言支持**：中文模式

**如何开发**：
1. 调用MinerU的混合自动引擎，传入PDF文件路径
2. 配置为中文语言模式，自动选择最佳解析方式
3. 输出Markdown格式文档
4. 实现缓存机制：检查是否已解析过，避免重复处理

**与需求结合**：
- 这是系统的入口流程，为后续所有核验环节提供数据基础
- 准确的PDF解析是保证后续核验准确性的前提

---

#### 流程二：MD → JSON 解析（规则解析器）

**技术手段**：
- **解析器**：规则引擎（无LLM依赖）
- **解析方式**：基于正则表达式和规则匹配
- **输出格式**：结构化JSON

**如何开发**：
1. 设计正则表达式匹配规则，提取证书元数据
2. 识别并解析表格结构，提取校准参数表
3. 输出包含两部分的JSON：
   - Meta元数据：仪器名称、型号、制造厂、证书编号、日期、环境信息等
   - Rows参数表：每个校准项目的参考值、示值、误差、不确定度
4. 实现缓存刷新机制：检测格式错误自动重新解析

**与需求结合**：
- 提取的Meta元数据服务于"证书类别"、"被测仪器信息完整性"、"环境条件"、"校准地点"、"日期选择"等需求
- 提取的Rows参数表服务于"校准项目、范围及不确定度合规性"等需求

---

#### 流程三：信息完整性核验

**技术手段**：
- **规则校验**：字段完整性检查
- **LLM增强**：DeepSeek API进行语义合理性核验
- **CNAS阻断**：非CNAS证书直接终止核验

**如何开发**：
1. 规则检查：验证必填字段是否存在（证书编号、仪器名称、型号、制造厂、温度、相对湿度、校准依据等）
2. CNAS阻断：检查是否为CNAS证书，若非CNAS直接终止
3. LLM语义核验：
   - 设计系统提示词，定义核验专家角色
   - 检查字段是否有乱码、空值
   - 验证仪器名称、型号、制造商逻辑一致性
   - 重点关注温度和相对湿度的合理性
4. 生成完整性核验报告

**与需求结合**：
- 直接对应需求："证书类别"、"被测仪器信息完整性"、"模板选择合规性"
- 间接支撑："结论选择合规性"的综合判定

---

#### 流程四：环境条件核验

**技术手段**：
- **向量检索**：Chroma DB检索温度要求数据库
- **语义匹配**：bge-m3嵌入模型计算相似度
- **LLM判断**：DeepSeek判断环境条件是否符合要求

**如何开发**：
1. 从JSON中提取当前温度、相对湿度、校准依据
2. 使用校准依据作为查询，在温度要求向量数据库中检索TopK条相关记录
3. 构建上下文：检索结果 + 当前环境条件
4. 调用LLM进行判断：
   - 设计环境核验专家的系统提示词
   - 输入当前环境条件和检索到的标准要求
   - LLM输出是否符合要求及详细对比说明
5. 生成环境条件核验报告

**与需求结合**：
- 直接对应需求："环境条件合理性"

---

#### 流程五：校准地点核验

**技术手段**：
- **向量检索**：Chroma DB检索认可的校准地点
- **双阈值匹配**：必须匹配阈值 + 可选匹配阈值
- **LLM增强**：判断地点具体性和合理性

**如何开发**：
1. 从JSON中提取校准地点数组
2. 使用地点向量进行检索，获取相似度最高的认可地点
3. 应用双阈值判断策略：
   - 必须匹配阈值：地点必须达到的最低相似度
   - 可选匹配阈值：更宽松的辅助判断标准
4. （可选）调用LLM判断地点描述的具体性和合理性
5. 生成校准地点核验报告

**与需求结合**：
- 直接对应需求："校准地点合理性"

---

#### 流程六：校准周期核验

**技术手段**：
- **双数据库查询**：华为数据库 + 通用周期数据库
- **默认规则**：默认校准周期12个月
- **日期逻辑**：检查校准日期、签发日期合理性

**如何开发**：
1. 从JSON中提取：建议校准周期、仪器名称、校准日期、签发日期
2. 优先查询华为周期数据库（针对华为设备）：
   - 使用仪器名称和型号进行向量检索
   - 若匹配到华为专用周期规则，应用之
3. 后备查询通用周期数据库：
   - 若华为库无匹配，检索通用周期库
   - 根据仪器类别匹配建议周期
4. 兜底规则：若两库均无匹配，默认12个月
5. 日期逻辑校验：
   - 验证校准日期 ≤ 签发日期
   - 检查时间差是否合理
6. 生成校准周期核验报告

**与需求结合**：
- 直接对应需求："日期选择合理性"

---

#### 流程七：参数与不确定度核验（最复杂）

**技术手段**：
- **向量检索**：Chroma DB检索CNAS校准能力数据库
- **语义理解**：推断参数量纲、匹配校准项目
- **单位换算**：自动换算单位，统一量纲
- **不确定度分析**：检查不确定度合理性
- **并发处理**：并行处理多个校准点

**如何开发**：
1. 解析JSON参数表，遍历每个校准项目
2. 对每个校准项目执行以下步骤：
   - **语义推断**：理解参数是什么（量纲、类型）
   - **向量检索**：用项目名称和仪器信息检索CNAS数据库，找最匹配的校准依据
   - **依据选择**：从检索结果中选择最匹配的CNAS认可的校准规范
   - **单位解析**：提取数值和单位，支持科学计数法
   - **单位换算**：统一到标准单位，支持倍数前缀（T、G、M、k、m、u、n、p等）
   - **范围检查**：验证测量值是否在CNAS认可的测量范围内
   - **不确定度检查**：验证证书给出的不确定度是否合理
3. 并发加速：使用线程池并行处理多个校准点
4. 生成详细的参数核验报告，记录每个参数的核验结果

**与需求结合**：
- 直接对应需求："依据选择合规性"、"校准项目、范围及不确定度合规性"、"标准器选择合理性"、"每个校准点结论合规性"
- 间接支撑："结论选择合规性"的综合判定

---

#### 流程八：报告生成与输出

**技术手段**：
- **Markdown格式**：易读易编辑的报告格式
- **模板渲染**：模块化报告拼接
- **结果汇总**：整合各模块核验结果

**如何开发**：
1. 收集各核验模块的报告
2. 按照标准格式拼接成完整报告
3. 总结整体核验结论：通过/不通过
4. 保存为Markdown文件
5. 提供下载和预览功能

**与需求结合**：
- 直接对应需求："核验签字"、"签发环节"

---

### 3.3 统一配置管理

**技术手段**：
- **不可变配置对象**：保证配置安全
- **环境变量支持**：支持12-Factor应用
- **方法链API**：流畅的配置覆盖方式

**如何开发**：
1. 定义配置类，包含所有配置项：
   - API配置：API Key、API地址、模型名称、参数
   - 路径配置：嵌入模型路径、各向量数据库路径
   - 业务配置：默认周期、是否启用LLM核验、匹配阈值
   - 目录配置：各缓存目录、输出目录
2. 实现从环境变量加载配置的功能
3. 提供配置覆盖方法，支持运行时修改部分配置
4. 实现目录自动创建功能

---

### 3.4 流水线编排

**技术手段**：
- **Hook机制**：回调接口，用于状态通知
- **进度跟踪**：实时显示核验进度
- **检查计划**：统一编排各核验模块
- **提前终止**：支持快速失败和用户中断

**如何开发**：
1. 定义Hook接口，包含：
   - 设置状态文本
   - 设置进度百分比
   - 信息、警告、错误、成功日志
2. 设计检查计划，按顺序编排各核验模块
3. 实现执行循环：
   - 遍历检查计划
   - 调用每个检查器
   - 更新状态和进度
   - 检查是否需要提前终止
4. 整合各模块报告，生成最终报告

## 四、向量数据库搭建

### 4.1 向量数据库总览

系统包含5个独立的向量数据库，分别用于不同的核验场景：

| 数据库名称 | 数据来源 | 用途 |
|-----------|---------|------|
| cnas_calibration | CNAS认可的校准能力范围 | 参数与不确定度核验 |
| temperature | 温度要求.xlsx | 环境条件核验 |
| general_cycle | 通用建议校准周期.xlsx | 校准周期核验 |
| huawei_cycle | 华为建议校准周期.xlsx | 校准周期核验（华为设备） |
| address | 校准地点.xlsx | 校准地点核验 |

---

### 4.2 CNAS 校准能力数据库搭建

**数据来源**：CNAS PDF原始文件

**数据预处理流程**：
1. PDF转TXT：提取PDF文本内容
2. HTML表格解析：识别并解析表格结构
3. Markdown解析与数据清洗：提取结构化数据
4. 构建向量数据库

**关键技术点**：
- **PDF解析**：使用pdfplumber + pdfminer双引擎
- **表格策略**：多套配置候选，自动选择最佳
- **科学计数法修正**：正确处理指数表示
- **正则表达式提取**：提取规范编号、日期、不确定度、测量范围
- **文本节点构造**：将每条校准能力封装为自然语言文本
- **嵌入模型**：BAAI/bge-m3生成语义向量
- **向量数据库**：Chroma DB存储和检索

---

### 4.3 其他数据库搭建

**温度要求数据库**：
- 数据来源：Excel文件
- 解析字段：校准依据、温度要求范围、湿度要求范围、特别说明
- 文档构造：自然语言句子形式

**校准地点数据库**：
- 数据来源：Excel文件
- 解析字段：序号、校准地址、专业室
- 文档构造：自然语言句子形式，便于语义检索
- 唯一ID：基于序号+地址+专业室的hash

**通用周期数据库**：
- 数据来源：Excel文件
- 解析字段：序号、细类名称、中类名称、大类名称、依据、建议校准周期
- 文档构造：包含仪器类别层级信息

**华为周期数据库**：
- 数据来源：Excel文件
- 特点：重点强调型号规格字段
- 文档构造：专门针对华为设备优化

## 六、关键技术亮点

### 6.1 缓存机制
- **三级缓存**：local_md → local_json → reports
- **智能刷新**：检测JSON格式错误自动重解析
- **提升效率**：避免重复OCR和LLM调用

### 6.2 向量检索 + LLM混合判断
- **快速筛选**：向量检索TopK快速缩小范围
